# Engineering Principles — 기술 리더의 의사결정 원칙

> **"불완전한 원칙이라도, 원칙이 없는 것보다 언제나 낫다."**

기술은 변한다. 도구는 변한다. 하지만 좋은 판단의 기준은 변하지 않는다.

이 책은 기술 리더가 매일 마주하는 수백 개의 판단 — 기술 선택, 인력 배치, 속도와 품질의 균형, 조직 구조 설계 — 에 대한 **판단의 운영 체제**를 제공한다.

## 구조

```
Part I.   내가 어디에서 왔는가        — 왜 이 원칙들이 나왔는가
Part II.  엔지니어링 원칙 (원칙 1~5)  — 개인의 판단 기준
Part III. 조직 원칙 (원칙 6~13)       — 조직의 판단 기준
Part IV.  원칙을 적용하는 법          — 실전 적용
성찰                                  — 마무리
부록                                  — 원칙 요약표
```

## 13개 원칙 요약

### 엔지니어링 원칙

| # | 원칙 | 핵심 질문 |
|---|------|-----------|
| 1 | 현실을 직시하고 대응하라 | 데이터에 기반한 판단인가, 감에 기반한 판단인가? |
| 2 | 다섯 단계 프로세스로 문제를 해결하라 | 해결책부터 찾고 있지 않은가? |
| 3 | 변하는 것과 변하지 않는 것을 구별하라 | 기술에 매료된 것인가, 문제에 집중한 것인가? |
| **4** | **이해하지 못하는 것은 배포할 수 없다 (관통 원칙)** | **이 결과물을 완전히 이해하고 있는가?** |
| 5 | 판단을 체계화하라 | 이 결정의 근거를 설명할 수 있는가? |

### 조직 원칙

| # | 원칙 | 핵심 질문 |
|---|------|-----------|
| 6 | 급진적 투명성 위에서 판단하라 | 나쁜 소식이 빨리 올라오는 구조인가? |
| 7 | 실수로부터 배우되 반복을 허용하지 마라 | 같은 실패가 반복되고 있지 않은가? |
| 8 | 자율과 책임의 균형을 설계하라 | 규칙이 구조적으로 강제되고 있는가? |
| 9 | WHO가 WHAT보다 중요하다 | 문제를 설계하는 사람이 올바른 자리에 있는가? |
| 10 | 지속적으로 훈련하고 성장시켜라 | 평가 기준이 시대에 맞게 재정의되었는가? |
| 11 | 조직을 기계로 바라보고 개선하라 | 조직의 병목이 어디인지 알고 있는가? |
| 12 | 모든 사람을 빌더로 만들어라 | 소수의 전문가에게만 의존하고 있지 않은가? |
| 13 | 도구와 프로토콜로 원칙을 구조화하라 | 원칙이 의지가 아닌 시스템으로 지켜지고 있는가? |
